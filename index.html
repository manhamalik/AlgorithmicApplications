Data Classes

Factorial Calculator
Linked Recursion

For this factorial calculator project, to implement the factorial calculations, linked recursion was used. The `calculate_factorial` function recursively calculates the factorial of a given number by breaking it down into smaller subproblems. It starts with the base cases of 0 and 1, returning 1. For any other number `n`, it recursively calls itself with `n - 1` and multiplies the result by `n`. This linked recursion approach allows for the efficient computation of factorials by leveraging the recursive nature of the factorial formula. The project provides a web interface where users can input a number, and it calculates and displays the factorial along with the step-by-step calculations. It enables users to quickly obtain factorials of various numbers, demonstrating the usefulness of linked recursion in performing repetitive mathematical computations effectively.

Event Scheduler
Binary Search Trees

For this event scheduler project, binary search trees (BSTs) were used to efficiently manage and organize events. The BST data structure allows for the insertion, retrieval, and sorting of events based on their date. The project utilizes the Event, Node, and EventScheduler classes. The Event class represents individual events with attributes such as date, time, location, and description. The Node class represents a node in the BST and holds an event object. The EventScheduler class manages the BST and provides methods for adding events recursively, performing an inorder traversal to retrieve events in sorted order, and maintaining the BST property. By utilizing BSTs, the project enables effective event scheduling, facilitates sorting and retrieval of events based on their dates, and provides an organized system for managing and displaying events.

Journal Entries
Hashing

For this journal entries project, hashing was used to create a HashTable data structure for efficient storage and retrieval of journal entries based on their dates. The HashTable class utilizes a dictionary (self.entries) to store entries where the date serves as the key. The insert method hashes the date and stores the entry in the dictionary using the hashed value as the key. Similarly, the remove method uses the date to hash and remove the corresponding entry from the dictionary. The find method retrieves the entry associated with a given date from the dictionary using hashing for quick lookup. By leveraging hashing, the project enables fast and constant-time access to journal entries based on their dates, improving the efficiency of adding, deleting, and viewing entries. The HashTable data structure provides a reliable and organized storage mechanism for journaling, facilitating easy management and retrieval of entries based on specific dates.
